Internet Services Browsers DNS HTML Tags

What is the Internet?

The Internet is a global network of interconnected computers or networks.

How the Internet Works

  1. User enters a URL: A user types a web address (URL) into their browser's address bar.
  2. DNS translation: The browser contacts a DNS server to translate the domain name into an IP address that identifies the web server.
  3. Browser sends request: The browser sends an HTTP/HTTPS request to the server at that IP address, asking for the webpage.
  4. Server processes request: The web server receives the request and retrieves the requested resources.
  5. Server sends response: The server sends the webpage data back to the browser via HTTP/HTTPS with a related code.
  6. Browser renders page: The browser receives the data and renders and display the webpage.

Main Internet Services

  • World Wide Web
  • File Transfer
  • video conferencing
  • Cloud Storage and Backup
  • email and Messaging Services

Daily Life Usage

Communication, education, entertainment, commerce, banking and government services.

Role of Web Browsers

Browsers and render HTML, CSS and JavaScript files.

What is DNS?

DNS translates domain names into IP addresses thatb are used to locate web servers.

HTML Tags Reference

Tag Purpose When to Use Example
<html> Root element that contains all HTML content on the page. Use once per document to add all HTML markup. <html></html>
<head> Container for page metadata, title, links, and scripts. Use inside <html> to define document metadata. <head></head>
<title> Defines the page title shown in the browser tab. Use inside <head> to set the document title. <title></title>
<meta> Provides metadata about the page such as charset or description. Use inside <head> for page settings and SEO information. <meta>
<link> Links external resources like stylesheets and icons. Use inside <head> to attach CSS or favicons. <link>
<section> Defines a thematic section of content. Use to group related content in a page. <section></section>
<body> Holds all visible content that appears in the browser. Use once per document to wrap page content. <body></body>
<header> Defines introductory content or site header area. Use for page headers or section headers. <header></header>
<nav> Groups navigation links to other pages or sections. Use for site menus and navigation blocks. <nav></nav>
<main> Represents the main content of the document. Use once per page for primary content. <main></main>
<article> Represents a self-contained composition like an article or post. Use for blog posts, news items, or independent content blocks. <article></article>
<aside> Contains content related to the main content, such as a sidebar. Use for side notes, ads, or related links. <aside></aside>
<footer> Defines footer content for a section or page. Use for copyright, links, and contact information. <footer></footer>
<h1> Top-level heading for a page or section. Use for the most important title on the page. <h1></h1>
<h2> Second-level heading in document structure. Use for major sections under <h1>. <h2></h2>
<h3> Third-level heading. Use for subsections under <h2>. <h3></h3>
<h4> Fourth-level heading. Use for smaller subsections in a hierarchy. <h4></h4>
<h5> Fifth-level heading. Use for lower-level section titles. <h5></h5>
<h6> Smallest heading level. Use for minor section headings. <h6></h6>
<description> Provides a description for a term or concept. Use for defining terms in a description list. <description></description>
<style> Defines CSS styles for the document. Use to apply visual formatting to elements. <style></style>
<img> Embeds an image in the page. Use for pictures, icons, and other visual content. <img>
<figure> Groups media and its caption together. Use around images, charts, or illustrations with captions. <figure></figure>
<caption> Provides a caption for a <figure>. Use inside <figure> to describe the media. <caption></caption>
<ul> Creates an unordered list of items. Use for lists where order is not important. <ul></ul>
<ol> Creates an ordered list of items. Use when item order or sequence matters. <ol></ol>
<li> Represents a list item. Use inside <ul>, <ol>, or <menu> elements. <li></li>
<dl> Defines a description list of terms and definitions. Use when pairing terms with descriptions. <dl></dl>
<dt> Defines a term/name in a description list. Use inside <dl> before a definition. <dt></dt>
<dd> Defines the description of a term. Use inside <dl> after a <dt>. <dd></dd>
<table> Creates a table for tabular data. Use to display rows and columns of structured data. <table></table>
<tr> Defines a row in a table. Use inside <table> to group table cells. <tr></tr>
<th> Defines a header cell in a table. Use inside a table row for column or row headings. <th></th>
<td> Defines a standard cell in a table row. Use inside <tr> for table data. <td></td>
<thead> Groups table header rows. Use to add heading rows in a table. <thead></thead>
<tbody> Groups the main table rows. Use to add the body rows in a table. <tbody></tbody>
<tfoot> Groups footer rows in a table. Use to add summary or total rows in a table. <tfoot></tfoot>
<form> Container for form controls and user input. Use to collect and submit data from users. <form></form>
<input> Defines an input control for text, choices, or buttons. Use inside a form to collect user data. <input>
<label> Defines a caption for a form control. Use to label inputs and improve accessibility. <label></label>
<num> Defines a numeric input field. Use when users need to enter a number. <input type="number">
<font> Defines the font and size of text. Use to style text with different fonts and sizes. <font></font>
<option> Defines an option inside <select>. Use to provide choices in a dropdown menu. <option></option>
<p> Defines a paragraph of text. Use to structure and organize textual content. <p></p>
<fieldset> Groups related form controls together. Use to organize fields in larger forms. <fieldset></fieldset>
<legend> Provides a caption for a fieldset. Use inside <fieldset> to label grouped controls. <legend></legend>
<datalist> Contains a list of predefined suggestions. Use with <input> for autocomplete options. <datalist></datalist>
<output> Displays the result of a calculation or user action. Use in forms or scripts to show output values. <output></output>
<div> Generic block container for grouping content. Use for layout and styling when no semantic tag fits. <div></div>
<span> Generic inline container for text or elements. Use for styling parts of text without changing layout. <span></span>
<strong> Indicates strong importance, typically rendered bold. Use to emphasize important text semantically. <strong></strong>
<em> Indicates emphasis, usually rendered italic. Use to emphasize a word or phrase in text. <em></em>
<mark> Highlights text for reference or emphasis. Use to mark important terms or search results. <mark></mark>
<small> Represents fine print or less important text. Use for disclaimers, legal text, or captions. <small></small>
<sup> Displays text as superscript. Use for exponents, footnotes, or trademarks. <sup></sup>
<sub> Displays text as subscript. Use for chemical formulas or mathematical indices. <sub></sub>
<code> Represents inline code or code fragments. Use for programming examples or command text. <code></code>
<select> Represents a dropdown list for selecting an option. Use for providing a list of choices to the user. <select></select>
<button> Represents a clickable button. Use for triggering actions or submitting forms. <button></button>
<q> Represents a short inline quotation. Use for short quotes within text. <q></q>
<textarea> Defines a multi-line text input field. Use for user comments, descriptions, or any editable text area. <textarea></textarea>
<address> Contains contact information for an author or organization. Use for addresses, phone numbers, and contact details. <address></address>
<time> Represents a date or time value. Use for timestamps, event dates, or deadlines. <time></time>
<audio> Embeds audio content with controls. Use for music, sound clips, and spoken content. <audio></audio>
<video> Embeds video content with controls. Use for movies, demos, and video clips. <video></video>
<source> Defines a media source for audio or video elements. Use inside <audio> or <video> to provide file sources. <source>
<iframe> Embeds another webpage inside the current page. Use to display external content like maps or widgets. <iframe></iframe>
<canvas> Defines a drawable region for graphics via scripting. Use for dynamic drawings, games, or visualizations. <canvas></canvas>
<svg> Defines scalable vector graphics. Use for icons, illustrations, and graphics that scale cleanly. <svg></svg>
<details> Creates a collapsible disclosure widget. Use for optional content that users can show or hide. <details></details>
<summary> Provides a caption for a <details> element. Use inside <details> to label the toggle. <summary></summary>
<progress> Represents task progress in a progress bar. Use to show how much of a task is complete. <progress></progress>
<meter> Represents a scalar measurement within a range. Use for values like battery level or temperature. <meter></meter>
<tel> Defines a telephone number input field. Use for collecting phone numbers from users. <tel></tel>
<a> Creates a hyperlink to another page or resource. Use to navigate to other pages or download files. <a href="url">Link Text</a>
<br> Inserts a line break in text. Use within text to force a new line. <br>
<hr> Creates a thematic break between content. Use to separate sections or topics in visual structure. <hr>
<b> Displays text in bold without added emphasis. Use for keywords or labels. <b></b>
<i> Displays text in italics without emphasis. Use for terms, foreign words, or scientific text. <i></i>
<u> Underlines text for presentation. Use for link-like styling or special emphasis. <u></u>
<del> Shows deleted or removed text. Use to mark text that is no longer current. <del></del>
<ins> Shows inserted or added text. Use to mark new or updated content. <ins></ins>
<marquee> Represents a scrolling text element. Use for animated text that moves across the screen. <marquee></marquee>
<samp> Represents sample output from a program. Use for example computer output or results. <samp></samp>
<var> Represents a variable or placeholder name. Use for code variables or values in text. <var></var>
<header> Represents the header of a section or page. Use to define the introductory content of a section. <header></header>
